# UI/动效风格模板（玻璃拟态 + 柔和渐变）

目标：统一“玻璃拟态 + 柔和渐变 + 轻量动效”的视觉语言，保持信息密度不降低、可读性优先。

## 视觉基调
- 氛围：清透、轻盈、柔和、有层次。
- 背景：大面积柔和渐变，避免纯色底。
- 卡片：半透明 + 细边框 + 轻阴影，形成“悬浮玻璃”感。
- 对比：文字颜色稳重，确保在半透明底上可读。

## 颜色与材质（参考变量）
- GlassWhite: 80% 白，用于主卡片底。
- GlassWhiteWeak: 40% 白，用于弱层级/子卡片。
- GlassBorder: 30% 白，用于描边。
- GlassShadow: 10% 黑，用于细阴影。
- 背景渐变示例：浅蓝 -> 浅绿 -> 柔黄。

## 布局与尺寸
- 页面内边距：16dp。
- 卡片圆角：24dp（大卡），16dp（小卡/按钮容器）。
- 卡片内部间距：10–12dp。
- 组件间距：12dp 主间距，8dp 次间距。
- 按钮：默认圆角 16–24dp。

## 组件样式
- 玻璃背景：全屏渐变背景。
- 玻璃卡片：半透明 + 细边框 + 轻阴影。
- 文本层级：
  - TitleLarge：页面标题。
  - TitleMedium：卡片标题。
  - BodySmall：说明与提示。
- 状态色：错误信息用明显红色，但保持整体柔和（降低饱和度）。

## 动效规范
- 展开/收起：Fade + Scale（0.95 -> 1.0）。
- 动效曲线：Spring (StiffnessLow)。
- 关键动作（展开、打开面板、弹出卡片）使用轻量弹性，避免过度晃动。
- 反馈：必要时结合轻震动提示（Haptic）。

## 交互原则
- 重要动作集中在卡片内，不漂浮过多控件。
- 视觉开关：允许用户关闭玻璃/模糊效果以提升性能。
- 文本优先清晰，玻璃效果不能遮挡可读性。

## 适配原则
- 手机：卡片纵向堆叠，留出呼吸感。
- 平板：卡片保持宽阔留白，避免满屏堆叠导致压迫。
